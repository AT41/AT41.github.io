Diet-Pi Emulator

With the current supply chain issues, chip shortages, and an increase in progamming hobbyists mainly caused by the current crisis *cough cough it can be hard to get the hardware required to start projects. When your desired Pi Hardware finally becomes available, you can port over what you've done so far - No need to rebuild your proejcts from scratch!

Since I wanted to take some screen shots for my digital photoframe tutorial I managed to get a working emulator up and running. Here's the steps I took to get it all set up.

Steps to Install

We'll be running QEMU: A generic and open machine emulator and virtualizer. Download QEMU for your respective OS (I'm running 64 bit Windows), and run through the installer.

Once that's done, navigate to where qemu was installed at. There should be a list of different qemu applications for different architects, but the one we're interested in is qemu-system-arm.

Our next step is to get the image, kernel, and dtb file for qemu-system-arm.

You can download the Diet-Pi image at DietPi.com. The image contains the operating system for the Pi as well as any data we store in it, such as pictures. For this tutorial choose the ARMv6 32-bit image, which is the DietPi version for the Raspberry Pi Zero.

Next we need the kernels and dtb files. You could theoretically download and build straight from the Linux repository on GitHub, but that was too much of a hassle on my Windows machine so I got everything from the dhruvvyas90's qemu-rpi-kernel repository on GitHub, and I highly recommend you to do the same! To emulate the software for the Raspberry Pi (2,3,4) and Zero models, download `kernel-qemu-4.14.79-stretch` and `versatile-pb-bullseye-5.10.63.dtb`. From here run the following command: `qemu-system-arm.exe -machine versatilepb -m 256 -cpu arm1176 -serial stdio -hda yourImageHere -kernel kernel-qemu-4.14.79-stretch -dtb versatile-pb-bullseye-5.10.63.dtb -append "root=/dev/sda2 rootfstype=ext4 rw panic=1" -no-reboot` Although the versatilepb machine won't have the same specs as the hardware, you should be able to use the image in the aforementioned Pis.

Personal note: I wasn't able to get the raspi0 hardware configuration properly running since the kernels and dtb files I extracted from the Diet-Pi image didn't work, but if you were able to get them to work I would love to hear about it!
